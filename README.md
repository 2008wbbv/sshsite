# ben vaccaro — ssh terminal site

Personal terminal accessible via SSH, built with [Charm wish](https://github.com/charmbracelet/wish).

```
ssh ben@your-app.up.railway.app -p 2222
```

## deploy to railway

### 1. install prerequisites

```bash
# install Go (if you don't have it)
# macOS:
brew install go

# install Railway CLI
npm install -g @railway/cli
```

### 2. get dependencies

```bash
cd ssh-site
go mod tidy   # downloads go.sum and all deps
```

### 3. push to github

Railway deploys from a git repo.

```bash
git init
git add .
git commit -m "init"
gh repo create ben-ssh-site --private --push --source=.
# or manually push to github.com
```

### 4. deploy on railway

```bash
railway login          # opens browser to authenticate
railway init           # creates a new project, select "Empty project"
railway up             # deploys from current directory
```

### 5. expose TCP port 2222

Railway doesn't auto-expose TCP ports — you need to do this in the dashboard:

1. Go to **railway.app/dashboard**
2. Open your project → your service
3. Click **Settings** → **Networking**
4. Click **Add TCP Proxy**
5. Set internal port to **2222**
6. Railway gives you a hostname like `roundhouse.proxy.rlwy.net` and a random external port like `12345`

### 6. connect

```bash
ssh ben@roundhouse.proxy.rlwy.net -p 12345
```

### 7. optional: clean domain

If you own a domain (e.g. benvaccaro.dev), add a CNAME:
```
ssh.benvaccaro.dev  →  roundhouse.proxy.rlwy.net
```
Then users can do:
```bash
ssh ben@ssh.benvaccaro.dev -p 12345
```

## local dev

```bash
go run .
# then in another terminal:
ssh localhost -p 2222
```
